## Astro 블로그 개발기 - Astro 시작하기 (LLM을 이용한 바이브 코딩)

### [ 부제 : Velog 스타일의 깔끔한 블로그를 Astro로 구현하기 ]

#### 1. 들어가기 앞서
안녕하세요. **zzado**입니다.
오늘 다룰 주제는 **Astro로 블로그 만들기**이다.

기존의 `Velog`나 `Tistory` 같은 좋은 플랫폼들이 이미 많이 존재한다. (나 또한 Velog를 잘 사용하고 있다.)
하지만 개발자라면 누구나 한 번쯤은 **"내 마음대로 커스터마이징 가능한 나만의 블로그"**를 꿈꾸게 된다.
특히 Github을 통해 글을 관리하면 **버전 관리**도 되고, 언제든 원하는 대로 마이그레이션이 가능하다는 점, 그리고 내가 구현하고 싶은 기능을 마음껏 추가할 수 있다는 점이 매력적으로 다가왔다.

얼마 전 **LLM을 이용한 바이브 코딩**이 인기여서, 공부도 할 겸 겸사겸사 직접 블로그를 만들어보게 되었다.
해당 포스팅에서는 **Next.js**나 **Gatsby** 같은 훌륭한 프레임워크들이 많음에도 불구하고 왜 **Astro**를 선택했는지, 그리고 어떤 과정을 거쳐 현재의 블로그를 완성했는지에 대한 기록을 남겨보려 한다.

#### 2. 기술 스택 선정 (왜 Astro인가?)
블로그를 만들기 위해 여러 프레임워크를 고민했다.
보통 React 기반의 블로그를 만든다면 다음과 같은 선택지들이 있다.

1.  **Next.js**: 가장 강력하고 대중적이다. 하지만 정적 블로그 하나를 만들기에는 다소 무겁게 느껴졌다. (물론 SSG도 훌륭하지만, JS 번들 사이즈가 기본적으로 좀 크다.)
2.  **Gatsby**: 플러그인 생태계가 엄청나다. 하지만 GraphQL 설정 등 러닝 커브가 있고, 빌드 속도 이슈가 종종 거론된다.
3.  **Astro**: **정답은 이것이었다.**

**Astro를 선택한 이유**는 명확하다.
-   **Content-First**: 콘텐츠 중심의 사이트에 최적화되어 있다. (블로그에 딱이다.)
-   **Zero JS by Default**: 꼭 필요한 부분(검색, 테마 토글 등)을 제외하고는 불필요한 자바스크립트를 배제하여 속도가 매우 빠르다.
-   **Islands Architecture**: React, Vue 등 원하는 프레임워크를 부분적으로 섞어 쓸 수 있다. (나는 검색 기능 등에 React 개념을 활용했다.)

#### 3. 개발 과정 및 주요 기능
개발 환경은 Mac OS이며, 패키지 매니저는 `npm`을 사용했다.

**3.1) 초기 설정 & 설치**
터미널에서 다음 한 줄이면 시작할 준비가 다 된 것이다.

```bash
npm create astro@latest
```

이후 몇 가지 설정을 마치면 기본적인 프로젝트 구조가 생성된다.
나는 여기에 **Tailwind CSS**를 추가하여 스타일링의 편의성을 높였다.

**3.2) 테마 시스템 구현 (다크 모드)**
개발자들이 가장 신경 쓰는 부분 중 하나인 **다크 모드**를 구현했다.
단순히 Light/Dark 만 있는 것이 아니라, 취향에 따라 선택할 수 있도록 4가지를 준비했다.
-   **Pure**: 깔끔한 기본 화이트
-   **Paper**: 눈이 편안한 미색 (Velog 느낌)
-   **Space**: 깊은 밤하늘 같은 남색
-   **Dark**: 표준 다크 모드

( `localStorage`를 이용해 사용자의 선택을 기억하고, 시스템 설정에 따라 자동으로 초기 테마를 잡아주는 기특한 기능도 넣었다. )

**3.3) 뷰 모드 토글 (List vs Card)**
사람마다 선호하는 보기 방식이 다르다.
누군가는 썸네일 위주의 **카드형**을 좋아하고, 누군가는 제목 위주의 **목록형**을 좋아한다.
이것도 `header`에 토글 버튼을 만들어 사용자가 선택할 수 있게 했다. (물론 이 설정도 저장된다.)

#### 4. 개인적인 생각 & 앞으로의 계획
**Astro**로 블로그를 만들어보니, 확실히 **"가볍다"**는 느낌을 많이 받았다.
불필요한 Hydration 과정이 없으니 페이지 로딩이 번개처럼 빠르다.
그리고 `.md` 또는 `.mdx` 파일로 글을 작성하면 되니, 마치 메모장에 글을 쓰듯 편안하게 포스팅을 할 수 있어 만족스럽다.

이제 뼈대는 완성되었다.
앞으로는 다음과 같은 기능들을 추가하거나, 그동안 공부했던 내용들을 하나씩 채워나갈 생각이다.
1.  **댓글 기능**: `Giscus` 등을 연동하여 소통 창구 만들기
2.  **시리즈 기능**: 연재 형식의 글을 묶어서 볼 수 있게 하기
3.  **GA 연동**: 방문자 통계 확인하기

혹시 나처럼 "나만의 블로그"를 고민하고 있다면, **Astro**를 강력하게 추천한다.
