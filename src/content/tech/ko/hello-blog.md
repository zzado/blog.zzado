## 1. 배경 (Why?)
안녕하세요. **zzado**입니다.
개발자라면 누구나 한 번쯤은 **"내 마음대로 커스터마이징 가능한 나만의 블로그"**를 꿈꾸게 됩니다.

기존의 `Velog`나 `Tistory` 같은 훌륭한 플랫폼들이 있지만, 다음과 같은 갈증이 있었습니다.
- **버전 관리**: Github을 통해 글과 코드를 안전하게 관리하고 싶다.
- **자유도**: 내가 원하는 기능(테마, 뷰 모드 등)을 마음껏 추가하고 싶다.
- **공부**: 특히 요즘 화제인 **LLM을 이용한 바이브 코딩**을 실전 프로젝트에 적용해보고 싶다.

그래서 직접 블로그를 만들어보기로 결심했습니다.

## 2. 기술 스택 선정 (Analysis)
블로그를 만들기 위해 React 기반의 프레임워크들을 저울질했습니다.

1.  **Next.js**: 가장 강력하고 대중적이지만, 정적 블로그 하나 만들기엔 다소 무겁고 설정이 복잡할 수 있습니다.
2.  **Gatsby**: 플러그인 생태계가 좋지만, GraphQL 등 러닝 커브가 있고 빌드 속도 이슈가 종종 언급됩니다.
3.  **Astro**: **결론은 이것이었습니다.** 🚀

**Astro를 선택한 이유:**
-   **Content-First**: 콘텐츠 중심의 웹사이트에 최적화되어 있습니다.
-   **Zero JS by Default**: 꼭 필요한 부분(상호작용)을 제외하곤 JS를 제거하여 **압도적인 성능**을 보여줍니다.
-   **Islands Architecture**: React 등 원하는 프레임워크를 부분적으로 섞어 쓸 수 있어 유연합니다.

## 3. 구현 과정 (Solution) 🛠️

### 3.1 초기 설정 & 설치
터미널에서 명령어 한 줄이면 프로젝트가 생성됩니다.
```bash
npm create astro@latest
```
여기에 **Tailwind CSS**를 붙여 스타일링 효율을 높였습니다.

### 3.2 테마 시스템 (Dark Mode) 구현
단순한 Light/Dark 모드를 넘어, 사용자의 취향을 존중하는 4가지 테마를 구현했습니다.
-   **Pure**: 깔끔한 화이트
-   **Paper**: 눈이 편한 미색 (Velog 스타일)
-   **Space**: 깊은 남색
-   **Dark**: 표준 다크 모드

`localStorage`를 활용해 사용자의 선택을 기억하도록 처리했습니다.

### 3.3 뷰 모드 토글 (List vs Card)
사용자마다 선호하는 UI가 다릅니다. 
- **Card View**: 썸네일 위주로 시원하게 보기
- **List View**: 제목 위주로 많은 글 훑어보기
이 두 가지를 버튼 하나로 전환할 수 있는 기능을 헤더에 탑재했습니다.

## 4. 결과 (Result) 📊
**Astro**로 만든 블로그는 확실히 **"가볍다"**는 느낌을 줍니다.
불필요한 Hydration 과정이 없으니 페이지 전환과 로딩이 번개처럼 빠릅니다.
또한 `.md` 파일로 글을 관리하니, 마치 코딩하듯 글을 쓸 수 있어 개발자 경험(DX) 측면에서도 매우 만족스럽습니다.
